# Iconography documentation

## Introduction
<p>
  <a href="https://www.figma.com/file/Hg1gtpuOUDdWaIeyy0pg90/%F0%9F%8F%B0-Ft.-Knox-Design-System?node-id=60:9633&t=SzJ2PJM44OKjZPSh-0">Axos Iconography</a> is a set of icons that are defined on the Axos Design System. The UI KIT is packed with all of those icons, for you to be able to use them in your application, without having to deal with the actual assets yourself.
</p>

### Constraints
<ol>
  <li>
    <strong>&lt;mat-icon&gt;</strong> supports displaying the icons by using the <strong>svgIcon</strong> property.
  </li>
  <li>
    <strong>Uk2IconRegistryService.registerAllCategories()</strong> method should be call from the top level component (app.component.ts) to register them in the <a href="https://v15.material.angular.io/components/icon/overview">MatIconRegistry from Angular Material</a>. Learn more about that their documentation. See the Prerequisites section below to know the correct import to use to access this service.
  </li>
  <li>
    The application should be configured properly to be able to access the SVG files from the UI KIT. See the Code Sample section to know more.
  </li>
</ol>

<br/>

## Sample Code

### Configuring assets access from angular.json file.
<p>angular.json</p>
<Source
  dark
  language='json'
  code={dedent`
    "[your-app-name]": {
      ...
      "architect": {
        "build": {
          ...
          "options": {
            ...
            "assets": [
              ...
              {
                "input": "node_modules/@axos/uikit-v2-lib/src/assets/svg/icons",
                "glob": "**/*",
                "output": "uk2/assets/svg/icons"
              }
            ]
          }
        }
      }
    }
  `}
/>

### Registering the icons from the top level component of your application.
<p>
  Note that registerAllCategories() method can be called from the constructor or the ngOnInit lifecycle hook.
</p>

<p>app.component.ts</p>
<Source
  dark
  language='typescript'
  code={dedent`
    import { Uk2IconRegistryService } from '@axos/uikit-v2-lib';
    ...
    export class AppComponent {
      constructor(private iconRegistryService: Uk2IconRegistryService) {
        this.iconRegistryService.registerAllCategories();
      }
    }
  `}
/>

### Basic usage

<p>
  Displaying an icon from Axos Iconography using mat-icon component.
</p>
<p>
  By default the icons will display in the same color as the surrounding text.
</p>

<p>*.component.html</p>
<Source
  dark
  language='html'
  code={dedent`
    <mat-icon [svgIcon]="alertsIcons.xCircle"></mat-icon>
  `}
/>

<p>*.component.ts</p>
<Source
  dark
  language='typescript'
  code={dedent`
    import { Uk2Tier1AlertsEnum } from '@axos/uikit-v2-lib';
    ...
    export class YourComponent {
      get alertsIcons() {
        return Uk2Tier1AlertsEnum;
      }
    }
  `}
/>

### Changing the color for Tier 1 and Tier 2 Mono icons
<p>
  Note the div element with <strong>colored-container</strong> css class, to apply a color.
</p>

<p>*.component.html</p>
<Source
  dark
  language='html'
  code={dedent`
    <div class="colored-container">
      <mat-icon [svgIcon]="alertsIcons.xCircle"></mat-icon>
    </div>
  `}
/>

<p>*.component.scss</p>
<Source
  dark
  language='css'
  code={dedent`
    .colored-container{
      color: yellow;
    }
  `}
/>

### Changing the color for Tier 2 Colored with Accent icons
<p>
  These icons have two elements. An element colored with the surrounding text color, and another element with an accent color (Primary / Blue 200 by default).
</p>
<p>
  To change the color of the surrounding text we can follow the previous code sample.
</p>
<p>
  To change the color for the accent part of the icons, we can overwrite the value for the css variable <strong>--uk2-icons-tier2-accent-color</strong>.
</p>

#### Overwriting the css variable to yellow from the main <i>styles.scss</i> file.
<p>*.component.html</p>
<Source
